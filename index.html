// core idea
Key insight: It's easier to merge two already-sorted lists than to sort one big unsorted one.
So we split down to single elements (trivially sorted), then merge back up using two pointers.
At every merge, always pick the smaller front element — this guarantees a sorted result every time.
01 // DIVIDE
Split in half recursively until every piece has 1 element. A single element is trivially sorted.
02 // CONQUER
Base case reached. Single elements need no sorting — recursion bottoms out and returns.
03 // MERGE
Two pointers walk sorted halves. Always take the smaller front. Result is guaranteed sorted.
04 // O(n log n)
log n levels × O(n) merge work per level. Holds for ALL inputs — no bad cases exist.

merge() — the code
function merge(arr, l, m, r) {
  let left  = arr.slice(l, m+1);
  let right = arr.slice(m+1, r+1);
  let i=0, j=0, k=l;
 
  while (i < left.length &&
         j < right.length) {
    if (left[i] <= right[j]) {
      arr[k++] = left[i++]; // take left
    } else {
      arr[k++] = right[j++]; // take right
    }
  }
 
  // append remaining elements
  while(i < left.length)  arr[k++]=left[i++];
  while(j < right.length) arr[k++]=right[j++];
}
